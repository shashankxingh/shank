Lecture 13: Arrays

Imagine you are making a game and need to store the scores of 100 players. Creating 100 separate variables (`score1`, `score2`, `score100`) would be a nightmare. When you need to store a collection of items that are all of the same data type, you use an Array.

1. Declaring and Initializing Arrays

An array is a single variable that holds a list of values. In Shank, an array is declared by adding square brackets [] after the data type.

# An immutable array of integers
let fibonacci: int[] = [0, 1, 1, 2, 3, 5, 8]

# A mutable array of strings
mut inventory: str[] = ["Sword", "Shield", "Potion"]

Important: In basic Shank, arrays are homogeneous. This means an int[] can ONLY hold integers. You cannot mix integers and strings in the same array.

2. Accessing Elements (Zero-Indexed)

You access individual items inside the array using their Index. In Shank, like in almost all programming languages, arrays are zero-indexed. This means the first item is at index 0, the second item is at index 1, and so on.

let colors: str[] = ["Red", "Green", "Blue"]

std.println(colors[0])  # Prints "Red"
std.println(colors[2])  # Prints "Blue"

3. Modifying Array Elements

If an array is declared with `mut`, you can change the values at specific indexes after the array has been created.

mut scores: int[] = [50, 60, 70]
scores[1] = 95  # Changes the 60 to a 95
std.println(std.format("Updated middle score: {}", scores[1]))

4. Iterating over Arrays

The true power of arrays comes when you combine them with loops. You can use a `for` loop to easily process every single element in an array without writing repetitive code.

let prices: float[] = [19.99, 5.50, 100.00]
mut total: float = 0.0

# Iterate directly over the items
for price in prices:
    total += price

std.println(std.format("Total cost is: {}", total))

5. Out of Bounds Errors

A very common beginner mistake is trying to access an index that doesn't exist. If an array has 3 elements (indexes 0, 1, 2), and you ask for index 3, the computer doesn't know what to do.

let sizes: int[] = [10, 20]

# RUNTIME ERROR! Index out of bounds. The program will crash.
# std.println(sizes[5])

Because Shank is a safe compiled language, it inserts boundary checks to intentionally crash the program if you do this, preventing serious memory corruption bugs.
