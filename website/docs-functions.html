Functions

Defining reusable blocks of code.

Defining a Function

Use def or fn to define a function. The two keywords are identical — pick whichever you prefer and stay consistent.

def greet(name: str):
    out "Hello, {name}!"

greet("world")

The body of a function is an indented block below the signature. There are no braces or end markers.

Parameter Types

Parameters can be typed in two styles:

Python style — type annotation after the name:

def add(a: int, b: int):
    return a + b

C style — type before the name:

def add(int a, int b):
    return a + b

If you omit the type, it defaults to int:

def double(x):
    return x * 2

Functions support up to 4 parameters.

Return Values

Use return to return a value from a function. The return type is inferred by the compiler from the return expression.

def square(n):
    return n * n

out square(7)    # prints 49

You can explicitly declare the return type with ->:

def square(n) -> int:
    return n * n

Functions without a return statement return none.

Forward Calls

Functions can be called before they are defined in the file. The compiler resolves all function signatures in a first pass.

out square(5)    # works

def square(n):
    return n * n

Functions and Top-Level Code

Functions and top-level statements can be mixed freely. Top-level code runs in order; function definitions are hoisted.

out add(2, 3)

im int x = 10
out add(x, 5)

def add(a, b):
    return a + b
Note: Recursive functions work, but the compiler does not yet detect infinite recursion. Be careful.
