Operators

Arithmetic, comparison, and logical operators.

Arithmetic Operators

Shank supports four arithmetic operators on numeric types.

Operator Description Example Result
+ Addition 5 + 3 8
- Subtraction 10 - 4 6
* Multiplication 6 * 7 42
/ Integer division 10 / 3 3
% Modulus (remainder) 10 % 3 1

Division truncates toward zero for integers. The result of 10 / 3 is 3, not 3.333.

Comparison Operators

All comparison operators return a bool value. Both operands must be the same type.

Operator Description
== Equal to
!= Not equal to
< Less than
> Greater than
<= Less than or equal to
>= Greater than or equal to 
im bool result = 5 > 3  // result is true

Logical Operators

Logical operators work exclusively on bool values. Using int or str operands is a compile-time error.

Operator Description
and Both operands must be true
or At least one operand must be true
not Unary; inverts a bool 
when x > 0 and x < 100:
    out "in range"

Operator Precedence

Operators are listed from highest to lowest precedence. Operators on the same row share the same precedence level.

Precedence Operators
Highest not
*   /   %
+   -
<   >   <=   >=
==   !=
and
Lowest or

Use parentheses to override the default precedence.

// Without parentheses: and binds tighter than or
im bool a = true or false and false   // true

// With parentheses: or evaluates first
im bool b = (true or false) and false  // false
