Operators & Math

An exhaustive deep-dive into Shank's arithmetic, logical operators, precedence climbing, and their direct mappings to x86-64 CPU instructions.

Unlike dynamic languages that use heavy runtime wrappers to handle math operations (like Python objects), Shank translates every operator into raw, single-cycle or multi-cycle CPU hardware instructions.

1. Arithmetic Operators

Arithmetic in Shank strictly adheres to type boundaries. You cannot perform an operation between an int and a float without an explicit -> cast.

Operator Name Types Supported Under-the-hood Assembly
+ Addition int, float add rax, rcx (int), addsd xmm0, xmm1 (float)
- Subtraction int, float sub rax, rcx (int), subsd xmm0, xmm1 (float)
* Multiplication int, float imul rax, rcx (int), mulsd xmm0, xmm1 (float)
/ Division int, float cqo; idiv rcx (int, truncates remainder), divsd (float)
% Modulus int only cqo; idiv rcx; mov rax, rdx (Fetches remainder from rdx)

Example: Modulus Constraints

Modulus (`%`) strictly targets `int` types to compute remainders. Attempting to use `%` on a `float` results in a fatal compiler error.

im int remainder = 10 % 3  # remainder is 1

2. Relational Operators

Relational operators compare two values of the exact same type and always return a bool (0 or 1). At the assembly level, these use the cmp instruction followed by `set*` condition codes.

  • == (Equality): Emits `sete al`
  • != (Inequality): Emits `setne al`
  • < (Less than): Emits `setl al`
  • > (Greater than): Emits `setg al`
  • <= (Less than or equal): Emits `setle al`
  • >= (Greater than or equal): Emits `setge al`
im bool is_adult = age >= 18

3. Logical Operators

Logical operators in Shank are built for speed. They enforce that both boundaries are strictly `bool` types, which allows the backend to utilize direct bitwise CPU operations instead of branching jumps, completely eliminating branch misprediction penalties on the CPU!

  • and: Logical AND. Compiles directly to and rax, rcx.
  • or: Logical OR. Compiles directly to or rax, rcx.
  • not: Logical NOT. Flips the boolean state using xor rax, 1.
im int x = 5
im bool valid = (x > 0) and (x < 10)
Why bitwise? Because Shank represents `true` as exactly 1 and `false` as exactly 0, an assembly-level `and` instruction successfully computes `(1 AND 0 = 0)`, creating an incredibly fast, non-branching evaluation.

4. Operator Precedence (Pratt Parsing)

Shank's compiler (`src/parser.c`) resolves math using a Recursive Descent Pratt Parsing algorithm. The precedence rules closely mirror mathematics (PEMDAS), determining how AST nodes are grouped before they are ever sent to code generation.

  1. Parentheses: ( expression ) (Highest)
  2. Logical NOT / Unary Minus: not, -
  3. Multiplication / Division / Modulus: *, /, %
  4. Addition / Subtraction: +, -
  5. Relational: ==, !=, <, >, <=, >=
  6. Logical AND: and
  7. Logical OR: or (Lowest)
im int result = 10 + 5 * 2 # Computes to 20, not 30!
