Standard Library & Built-ins

A reference manual for Shank's native runtime functions, I/O operations, and type coercion mechanisms.

Output Functions

Shank provides three highly optimized built-in keywords for streaming data to the standard output buffer. Unlike standard functions, these operate as native language keywords integrated directly into the parser.

out (Print with Newline)

The standard output mechanism. It prints a formatted representation of any primitive type (`int`, `float`, `bool`, `str`) and appends a newline (`\n`).

out "Compilation finished."

outt (Print Text Inline)

Identical to `out`, but it does not append a newline. Crucial for procedural text generation, such as printing grid matrices.

repeat 5:
    outt "*"
out "" // Flushes the line

outc (Print ASCII Character)

A specialized instruction that accepts a 64-bit integer and emits the corresponding single byte ASCII character directly to the standard output stream. Excellent for low-level formatting or procedural rendering without string allocations.

outc 65   // Prints 'A'
outc 10   // Prints a newline '\n'

Type Coercion (Casting)

Because Shank is strictly typed, operations between mismatched types (like concatenating a string and an integer) require explicit casts. The language exposes four global casting functions.

Function Description Example
str(value) Safely converts integers, floats, or booleans to a heap-allocated string pointer. str(42) // "42"
int(value) Parses a string into a 64-bit integer, or truncates a float to an integer. int("100") // 100
float(value) Parses a string into a 64-bit double, or elevates an integer to a float. float(5) // 5.0
bool(value) Evaluates truthiness (e.g. non-zero ints become true, empty strings become false). bool(1) // true

Operators & Arithmetic Constraints

Shank implements rigorous mathematical operator enforcement via the Type Checker (`src/checker.c`).

  • Arithmetic: `+`, `-`, `*`, `/`, `%` (Modulus). Note: Modulus operates strictly on `int` types.
  • Relational: `==`, `!=`, `<`, `>`, `<=`, `>=`. Relational operators return a `bool` and require identical types on both boundaries.
  • Logical: `and`, `or`, `not`. These operators accept only `bool` values, operating via heavily optimized bitwise assembly paths (`and rax, rcx`) without branching logic.
