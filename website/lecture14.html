Lecture 14: Strings

We've used Strings (`str`) throughout this course to print messages and store text. But Strings are more complex than basic integers or booleans. Under the hood, a string is actually a sequential collection—much like an Array—of characters. In this final core lecture, we will learn how to manipulate text data professionally.

1. Strings as Arrays of Characters

Because Strings are sequences, you can interact with them similarly to how you interact with Arrays. You can retrieve a specific character by accessing its index.

let word: str = "Shank"

# Get the first character (Index 0)
std.println(word[0])  # Prints "S"

# Get the last character
std.println(word[4])  # Prints "k"

However, unlike Arrays, Strings in Shank are immutable by default regarding their individual characters. You cannot do `word[0] = "T"`. If you want to change a string, you must build a completely new one.

2. String Concatenation

Concatenation is a fancy word for "gluing strings together." You can combine strings using the `+` operator. If you want to continuously append text to a mutable string, you can use the `+=` operator.

let first_name: str = "Grace"
let last_name: str = "Hopper"
let full_name: str = first_name + " " + last_name

mut log: str = "System Started. "
log += "Loading files... "
log += "Done."
std.println(log)

3. String Length

Often, you need to know exactly how many characters are in a string. For example, verifying that a password is at least 8 characters long. The standard library provides a function for this: `std.len()`.

let pass: str = "hunter2"

if std.len(pass) < 8:
    std.println("Error: Password is too short!")

4. Iterating Over Strings

Just like looping through an array, you can use a `for` loop to step through a string character by character. This is highly useful for tasks like counting vowels or searching for specific letters.

let text: str = "banana"
mut a_count: int = 0

for char in text:
    if char == "a":
        a_count += 1

std.println(std.format("Found {} 'a's.", a_count))

5. UTF-8 Encoding (Advanced)

It's important to note that Shank strings are UTF-8 encoded. Standard ASCII characters (like 'A' or '7') take up 1 byte of memory. However, special characters like Emojis (`😎`) or characters from non-Latin alphabets might take 2 to 4 bytes! Shank's `std.len()` safely handles this, counting the visual characters rather than the raw byte length.
