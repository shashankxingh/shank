Data Types & Coercion

An exhaustive deep-dive into Shank's type system, primitive memory sizes, and the casting operator (->).

Shank is a strictly, statically typed compiled language. Type enforcement happens purely at compile time during the AST semantic analysis phase (`src/checker.c`). By enforcing types before execution, Shank eliminates massive categories of runtime exceptions.

1. Primitive Data Types

Shank provides four fundamental primitive types, mapped directly to their corresponding raw x86-64 hardware registers or memory blocks. Each occupies exactly 8 bytes (a QWORD) on the stack, ensuring perfectly aligned memory accesses without complex padding algorithms.

The int Type

A 64-bit signed integer. Handled internally in general-purpose integer registers like `rax` and `rcx`.

  • Range: -9,223,372,036,854,775,808 to 9,223,372,036,854,775,807
  • Declaration: im int x = 42
  • Usage: Ideal for absolute quantities, math, indexing, and counters.

The float Type

A 64-bit IEEE 754 double-precision floating-point number. Handled internally in SSE/AVX registers like `xmm0` and `xmm1`.

  • Declaration: im float pi = 3.14159
  • Usage: Required for fractional arithmetic. Shank enforces a strict barrier between `int` and `float` calculations. You cannot blindly multiply an `int` with a `float` without explicit coercion.

The bool Type

A logical boolean indicating `true` or `false`. Even though a boolean only requires a single bit (and conceptually 1 byte, e.g., `al` register), Shank aligns it into an 8-byte stack block for seamless execution speed.

  • Values: true (maps to integer 1), false (maps to integer 0).
  • Declaration: im bool is_ready = true

The str Type

A 64-bit memory pointer pointing to a null-terminated UTF-8 sequence residing in the binary's data section (`.data`) or allocated heap.

  • Declaration: im str msg = "Welcome!"
  • F-Strings: Shank natively supports string interpolation via the f"" literal.
    im str greeting = f"Hello, {name}"

2. Type Coercion (The Arrow Operator)

Because Shank strictly isolates types, you cannot implicitly pass an `int` where a `str` is expected, or perform arithmetic between an `int` and a `float` without an explicit conversion boundary. This is where the highly specific Casting Arrow (->) comes into play.

Basic Syntax

You cast an expression by appending -> type_name to it.

im int age = 25
im str display = age -> str   # Converts 25 into "25"

Coercion Scenarios

The `src/checker.c` engine rigorously verifies your cast before allowing compilation to proceed. Valid casts include:

  • str -> int: Safely parses a string into a 64-bit integer. Useful when parsing terminal input.
    im int x = "100" -> int
  • str -> float: Parses a string into double-precision float.
    im float y = "3.14" -> float
  • int -> str / float -> str / bool -> str: Allocates memory and stringifies the underlying primitive for formatting or concatenating operations.
Invalid Cast Protections: Shank prevents nonsensical casts. Attempting to cast an arbitrary type to an undefined struct or failing to adhere to the primitive bounds will trigger a fatal `Invalid cast from 'x' to 'y'` error.

3. F-Strings (Interpolation)

Often, you need to combine integers, strings, and floats together. Instead of chaining multiple -> str casts, Shank integrates a dedicated TOK_FSTR_START lexical parser.

By simply prefixing your string with f, any expression inside `{}` is automatically evaluated and cleanly cast to a string by the backend.

im int score = 99
im str name = "Player One"

out f"Result: {name} achieved {score} points."
