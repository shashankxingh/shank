Type Casting

Converting values between types with the -> operator.

The -> Operator

Explicit type conversion is performed by placing -> between an expression and a target type name.

im float pi = 3.14159
im int rounded = pi -> int
out rounded     # prints 3

Supported Conversions

FromToBehavior
floatintTruncation toward zero. 3.7 -> int yields 3. -2.9 -> int yields -2.
intfloatExact conversion. 42 -> float yields 42.000000.
intstrDecimal string representation. 42 -> str yields "42".
floatstrDecimal string representation. 3.14 -> str yields "3.140000".
boolstrtrue -> str yields "true". false -> str yields "false".

Using Casts in Expressions

Casts can appear anywhere an expression is accepted.

im float price = 9.99
out "Whole part: {price -> int}"
im int a = 7
im int b = 2
im float result = a -> float / b -> float
out result       # 3.500000

Type Safety

Shank does not perform implicit type coercion. Assigning a float to an int variable without an explicit cast produces a compile-time error.

im int x = 3.14
# error: Type mismatch: expected 'int', got 'float'.

im int x = 3.14 -> int    # correct
