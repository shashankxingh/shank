Loops & Iteration

An exhaustive deep-dive into Shank's bounded `repeat` loops, conditional `while` loops, and iteration mutability.

Unlike dynamic languages that abstract loops over arbitrary objects via iterators, Shank handles looping strictly at the stack level. The compiler natively maps iteration logic directly to x86-64 unconditional jumps (`jmp`) and loop evaluators without relying on runtime garbage collection.

1. Bounded Iteration: repeat

The primary and safest way to loop in Shank is the `repeat` keyword. It provides mathematically bounded iteration over a strictly evaluated integer count.

Basic Repetition

If you don't need access to the loop's current iteration count, simply provide an `int` to `repeat`.

repeat 3:
    out "Connecting to server..."

Accessing the Index with as

Using the `as` keyword instantly binds the current iteration count (starting at 0) to a highly protected, scope-locked `im` variable.

repeat 5 as i:
    out f"Processing item {i}"
Memory Mutability: The loop variable i is Strictly Immutable. If you attempt to reassign i inside the block (e.g., i = 10), the compiler's semantic phase will instantly halt the build. You cannot accidentally skip loop ticks.

2. Conditional Iteration: while

When you do not know the boundary of a loop at compile time (such as polling a socket or waiting for input), Shank implements a strict `while` loop.

Syntax and Boolean Constraint

Exactly like `when` statements, the `while` statement requires a strict `bool` expression. Treating an `int` or a `str` as a condition will result in a fatal type error.

mut int battery = 100

while battery > 0:
    out f"Running on {battery}%"
    battery = battery - 10

out "Battery empty."

3. Under the Hood: Assembly Translation

The Code Generation phase (`src/codegen.c`) transforms loops into highly efficient register mathematics.

The while Loop Mechanics

A `while` loop translates into a top label (`.L_while_cond`) and a bottom jump. At the start of every iteration, the AST evaluates the condition into the `rax` register. If the boolean is `0` (`false`), the compiler emits a `je` (Jump if Equal) instruction to immediately bypass the entire block.

.L_while_cond_1:
    ; ... [Condition evaluation in rax] ...
    cmp rax, 0
    je .L_while_end_1
    
    ; ... [Block Execution] ...
    jmp .L_while_cond_1
.L_while_end_1:

The repeat Loop Mechanics

Because `repeat` requires a named index variable, the backend actually allocates the index variable onto the Stack Pointer (`rbp`) just like a regular `let` variable. It initializes it to 0, evaluates the upper bound, and increments it mathematically at the end of the block. If you use nested `repeat` loops to build 2D grid patterns, the nested indices do not collide because they have distinct stack offsets (e.g., `rbp-16` and `rbp-24`).
