Control Flow

Detailed guide to branching logic and looping mechanisms in Shank.

Conditional Branching

Shank provides a robust mechanism for conditionally executing code using the when, elif, and otherwise keywords.

The when Statement

A when statement requires a boolean expression followed by a colon. The indented block below it is executed only if the expression evaluates to `true`.

im int age = 18

when age >= 18:
    out "Access granted."

elif and otherwise

You can chain multiple conditions using elif (short for "else if"). If all preceding conditions fail, the otherwise block acts as the fallback execution path.

im int score = 85

when score >= 90:
    out "Grade: A"
elsewhen score >= 80:
    out "Grade: B"
elsewhen score >= 70:
    out "Grade: C"
otherwise:
    out "Grade: F"
Type Enforcement: Conditions in `when` and `elif` clauses must explicitly resolve to a `bool`. Trying to evaluate an integer like `when 1:` will trigger a compiler type error.

Looping Construct: repeat

Instead of traditional C-style `for` loops or `while` loops, Shank intentionally implements a specialized integer-bound loop via the repeat keyword. This enforces bounded iteration and prevents accidental infinite loops.

Basic Repetition

To execute a block of code a specific number of times, provide an integer value to `repeat`.

repeat 3:
    out "Processing..."

Accessing the Iteration Variable

You can bind the current iteration index to a local, immutable variable using the as keyword. The index starts at `0` and increments by `1` until it reaches `N - 1`.

repeat 5 as i:
    out "Iteration: {i}"

Iteration: 2

Iteration: 4

Nested Loops and Variable Scoping

When nesting `repeat` loops, iteration variables (`i`, `j`, etc.) are securely scoped to their respective indented blocks. They do not leak into the outer scope after the block terminates.

im int size = 3

repeat size as row:
    repeat size as col:
        when row == col:
            outt "*"
        otherwise:
            outt "-"
    out ""
