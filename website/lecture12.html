Lecture 12: Methods / Functions

As your programs grow larger, dumping hundreds of lines of code into a single file becomes unmanageable. Code gets messy, repetitive, and difficult to debug. To solve this, programmers break their code into smaller, reusable, named blocks. In Shank, these are called Functions.

1. What is a Function?

A function is a self-contained block of code that takes in some input, performs a specific action, and optionally returns an output. We have already been using functions like std.print() and the mandatory main() function. Now it is time to write our own.

2. Defining a Function

To define a function in Shank, you use the fn keyword, followed by the function's name, parentheses (), an arrow -> indicating the return type, and finally a colon :. The code inside the function must be indented.

fn say_hello() -> none:
    std.println("Hello from inside a function!")

In this example, the return type is none. This is a special keyword in Shank meaning the function just performs an action and does not send any data back to the caller.

3. Parameters (Passing Data In)

Functions are most useful when they can operate on different data. We pass data into a function using Parameters. These act as variables that are created the moment the function is called.

fn greet_user(name: str, age: int) -> none:
    std.println(std.format("Welcome, {}, you are {} years old.", name, age))

fn main() -> int:
    # Calling the function
    greet_user("Alice", 28)
    greet_user("Bob", 45)
    return 0

Because Shank is strictly typed, you must specify the type of every parameter (e.g., name: str).

4. Return Values (Passing Data Out)

Instead of just printing a result, functions usually calculate a value and return it so the rest of the program can use it. We use the return keyword for this.

fn add_numbers(a: int, b: int) -> int:
    let sum: int = a + b
    return sum

fn main() -> int:
    let total: int = add_numbers(10, 15)
    std.println(std.format("The total is {}", total))
    return 0

Notice that the return type is explicitly set to int. If you try to return a str from this function, the compiler will throw an error!

5. Variable Scope

Variables created inside a function are local to that function. They exist only while the function is running. Once the function finishes and returns, those variables are destroyed from the Stack memory.

fn secret_calculation() -> none:
    let secret_value: int = 42

fn main() -> int:
    secret_calculation()
    # ERROR! 'secret_value' does not exist here.
    # std.println(secret_value) 
    return 0
