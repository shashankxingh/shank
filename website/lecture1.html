Lecture 1: Flowchart & Pseudocode + Basics

Welcome to your very first step in learning the Shank programming language! Before we dive into writing code, it is absolutely essential to understand the logic behind programming. Programming is not just about memorizing syntax; it is about problem-solving. In this lecture, we will explore flowcharts, pseudocode, and the fundamental basics of the Shank language.

1. Introduction to Problem Solving

A computer is inherently a dumb machine. It only does exactly what you tell it to do. If you tell it to jump, it will jump. If you tell it to jump off a cliff, it will do so without hesitation. Therefore, your job as a programmer is to give the computer a series of incredibly precise, logical instructions. This process begins with problem-solving.

When faced with a programming challenge, you should never start by typing code. Instead, you should start by breaking the problem down into smaller, manageable steps. This is where Flowcharts and Pseudocode come in.

2. Flowcharts: Visualizing Logic

A flowchart is a visual representation of an algorithm or process. It uses standardized symbols to depict different types of actions or steps in a process. These symbols are connected by arrows showing the flow of control.

  • Oval (Terminal): Represents the Start or End of a program.
  • Parallelogram (Input/Output): Represents an input operation (e.g., getting data from the user) or an output operation (e.g., printing data to the screen).
  • Rectangle (Process): Represents a process, action, or calculation (e.g., adding two numbers).
  • Diamond (Decision): Represents a decision point where a condition is evaluated, leading to different paths (e.g., "Is the number greater than 10?").

By sketching out a flowchart, you can trace the execution path of your program and identify logical flaws before writing a single line of Shank code.

3. Pseudocode: Logic in Plain English

Pseudocode is a way of writing out a program's logic using plain English (or any other human language) mixed with some programming-like constructs. It is entirely independent of any specific programming language's syntax. The goal of pseudocode is to focus on what the program should do, not how the compiler expects it to be written.

Here is an example of pseudocode for a program that checks if a user is old enough to vote:

START
PRINT "Enter your age:"
INPUT age
IF age >= 18 THEN
    PRINT "You are eligible to vote."
ELSE
    PRINT "You are not eligible to vote yet."
END IF
END

4. Translating to Shank Basics

Once you have a solid flowchart and pseudocode, translating it into Shank is a breeze. Shank is designed to be highly readable, borrowing Python's indentation-based syntax while providing the performance of a compiled language like C.

Let's look at a basic Shank program. Every standalone Shank script begins execution from the top down. There is no boilerplate or mandatory main function needed!

# This is a comment in Shank. The compiler ignores it.

im str message = "Hello, World!"
out message

In the above example:

  • im str defines an immutable variable of type string.
  • out prints the variable to the screen natively.
  • Notice there are no semicolons, no curly braces, and no fn main(): wrapper! Shank is designed to be as clean as possible.

5. Common Mistakes for Beginners

  • Skipping the planning phase: Jumping straight into coding without a flowchart or pseudocode often leads to messy, unreadable code and logical errors.
  • Indentation errors: Since Shank relies on indentation (spaces or tabs) to determine code blocks, inconsistent indentation will cause the compiler to throw errors.
  • Forgetting types: Shank is statically typed. Even though it looks like Python, you must declare types (like : str) or allow the compiler to infer them properly.

6. Exercises

To reinforce what you've learned, try completing these exercises:

  1. Draw a flowchart for a process that makes a cup of tea. Include decisions (e.g., "Do I want sugar?").
  2. Write pseudocode for a program that asks the user for two numbers, adds them together, and prints the result.
  3. Translate your addition pseudocode into Shank. Try compiling and running it (we will learn how to run code in the next lecture!).
