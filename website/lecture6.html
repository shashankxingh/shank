Lecture 6: DataTypes

In the previous lecture, we learned that variables are containers for data. But data comes in many different shapes and sizes. A person's age is a whole number, their bank balance might have decimals, their name is a sequence of characters, and whether they are active or not is a simple true/false answer. In Shank, you must specify the DataType so the compiler knows exactly how to handle and store that information in memory.

1. Why Do Data Types Matter?

Shank is a strictly typed language. The compiler will literally stop you from doing things that don't make sense logically based on data types. For example, you cannot mathematically divide a name by a number. Defining types keeps your program safe, fast, and free of bizarre runtime crashes.

2. Core Data Types in Shank

Shank provides four fundamental primitive data types that serve as the building blocks for all your programs.

Integers (`int`)

The `int` type represents whole numbers without any fractional or decimal component. Under the hood, Shank uses 64-bit integers by default, which means you can store incredibly large numbers (both positive and negative).

let year: int = 2026
let temperature: int = -15
mut population: int = 8000000000

Floating-Point Numbers (`float`)

When you need precision that includes decimal points, you use the `float` type. Shank uses 64-bit floating-point numbers (often known as "doubles" in other languages). Use floats for measurements, currency, and precise calculations.

let pi: float = 3.14159
mut account_balance: float = 1050.75

Strings (`str`)

Text data is stored in the `str` type. A string is essentially a sequence of characters strung together. In Shank, strings are UTF-8 encoded, meaning they can support standard text as well as emojis. String values MUST be enclosed in double quotation marks " ".

let greeting: str = "Hello, World!"
mut user_name: str = "Shashank"

Booleans (`bool`)

The simplest data type is the `bool`. It can only hold one of two possible values: `true` or `false`. Booleans are the core of decision-making logic in programming. Note that in Shank, these keywords must be written in entirely lowercase.

let is_admin: bool = true
mut game_over: bool = false

3. Common Type Errors

Because Shank is strictly typed, mixing types inappropriately will result in a compiler error. This is a feature, not a bug!

let number: int = 5
let text: str = "apples"

# ERROR: Cannot add an integer and a string
# let result = number + text

If you need to mix types (like printing a number inside a sentence), you will have to convert them or use string formatting, which we will cover in a later lecture.
