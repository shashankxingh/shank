Variables

Declaring and using variables in Shank.

Immutable Variables — im

Variables declared with im cannot be reassigned after initialization. Use im by default; reach for mut only when reassignment is necessary.

im int age = 25
im str name = "Shashank"
im float pi = 3.14159
im bool active = true

Attempting to reassign an immutable variable produces a compile-time error:

im int x = 10
x = 20  // error: cannot reassign immutable variable 'x'

Mutable Variables — mut

Variables declared with mut can be reassigned freely after initialization. The new value must match the declared type.

mut int counter = 0
counter = counter + 1
counter = 99

Declaration Syntax

The full form of a variable declaration is:

<mutability> <type> <name> = <value>
  • mutability — either im (immutable) or mut (mutable).
  • type — one of int, float, str, bool.
  • name — the variable identifier.
  • value — the initial value. All variables must be initialized at declaration.
Every variable must have an explicit type. Shank does not support type inference for variables.
