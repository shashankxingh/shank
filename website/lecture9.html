Lecture 9: Conditionals

Up until now, our code has executed in a straight line, from top to bottom. But real-world logic requires making decisions. "If it is raining, take an umbrella, otherwise, wear sunglasses." Conditionals allow your Shank programs to branch out and execute different blocks of code depending on whether certain conditions are true or false.

1. The `if` Statement

The `if` statement is the most basic conditional. It evaluates a boolean expression (a statement that is either `true` or `false`). If the condition is true, the indented block of code underneath it runs. If it is false, the block is completely skipped.

let score: int = 85

if score >= 50:
    std.println("Congratulations, you passed!")

std.println("This line always prints.")

Crucial Rule: Shank does not use `{}` curly braces to group code. Instead, it uses a colon `:` and Indentation. Every line of code that belongs inside the `if` statement must be indented by exactly 4 spaces (or 1 tab). When the indentation returns to normal, the `if` block is over.

2. The `else` Statement

What if you want something to happen when the condition is false? You use the `else` keyword. It provides a fallback path.

let password: str = "1234"

if password == "secret":
    std.println("Access Granted.")
else:
    std.println("Access Denied!")

3. The `elif` Statement

Sometimes you have more than two possibilities. You can chain conditions together using `elif` (short for "else if"). The compiler checks them from top to bottom, and only the first true block will run.

let grade: int = 75

if grade >= 90:
    std.println("Grade: A")
elif grade >= 80:
    std.println("Grade: B")
elif grade >= 70:
    std.println("Grade: C")
else:
    std.println("Grade: F")

4. Nested Conditionals

You can put `if` statements inside other `if` statements. Just remember to increase your indentation for every new level!

let has_ticket: bool = true
let age: int = 15

if has_ticket:
    if age >= 18:
        std.println("Welcome to the movie.")
    else:
        std.println("You are too young for this movie.")
else:
    std.println("You need a ticket.")

5. The `match` Statement (Pattern Matching)

If you have a single variable that you want to compare against many exact values, long `elif` chains can get ugly. Shank provides a much cleaner `match` statement (similar to `switch` in other languages).

let command: str = "attack"

match command:
    case "attack":
        std.println("Swinging sword!")
    case "defend":
        std.println("Raising shield!")
    case "run":
        std.println("Running away!")
    case _:
        # The underscore _ is the default case (like else)
        std.println("Unknown command.")
