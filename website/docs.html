Shank Documentation

Language reference for Shank v0.1 — a statically typed, compiled language with Python-like syntax.

Overview

Shank compiles .sk source files directly to x86-64 machine code via NASM and GCC. The output is a standalone Windows .exe — no runtime or interpreter needed.

The compiler toolchain is invoked through the shank CLI:

# compile and run immediately
shank run myfile.sk

# compile to an executable
shank build myfile.sk -o myfile.exe

# type-check without compiling
shank check myfile.sk

Files use the .sk extension by convention.

Hello World

Shank programs execute top-level statements from top to bottom. There is no required main function, no imports, no boilerplate.

out "Hello, World!"

Save this as hello.sk and run it:

$ shank run hello.sk
Hello, World!

That single line is a complete, compilable Shank program. out prints a value followed by a newline.

Variables

Variables are declared with a mutability keyword, followed by a type and a name. Every variable must be initialized at declaration.

Immutable variables — im

Immutable variables cannot be reassigned after their initial value is set.

im int age = 25
im str name = "Shashank"
im float pi = 3.14159
im bool active = true

Attempting to reassign an im variable produces a compile-time error:

im int x = 10
x = 20     # error: Cannot assign to immutable variable 'x'.

Mutable variables — mut

Mutable variables can be reassigned freely.

mut int counter = 0
counter = counter + 1
counter = 99
out counter     # prints 99

Declaration syntax

The full form is:

<mutability> <type> <name> = <value>
PartOptions
Mutabilityim (immutable) or mut (mutable)
Typeint, float, str, bool
Note: Every variable must have an explicit type. Shank does not support type inference for variables in v0.1.

Data Types

Shank has four primitive types. All are statically resolved at compile time.

The int type covers the full 64-bit signed range: −9,223,372,036,854,775,808 to 9,223,372,036,854,775,807.

Strings support interpolation with {} — see String Interpolation.

Comparison

All comparison operators return a bool. Both operands must be the same type.

OperatorDescription
==Equal
!=Not equal
<Less than
>Greater than
<=Less than or equal
>=Greater than or equal
 
im bool result = 5 > 3
out result     # prints True

Logical

Logical operators only accept bool operands. Using int or str is a compile-time error.

OperatorDescription
andTrue if both operands are true
orTrue if at least one operand is true
notInverts a bool (unary)
 
im int x = 50
when x > 0 and x < 100:
    out "in range"

Use parentheses to override: (a + b) * c

Output — out

out prints a value to standard output followed by a newline. It works with all four types natively.

out 42            # int
out "hello"       # str
out 3.14          # float
out true          # bool

You can print the result of any expression:

im int x = 10
out x * 2         # prints 20
out x > 5         # prints True

Works with interpolated strings too:

im str name = "Shashank"
out "Hello, {name}!"

Input — put()

put() prints a prompt string, reads a line from stdin, and returns the value parsed as the type of the receiving variable.

im int age = put("Enter your age: ")
im str name = put("Enter your name: ")
im float height = put("Height in meters: ")
im bool agree = put("Agree? (true/false): ")

The compiler determines which runtime input function to call based on the declared type of the receiving variable.

Invalid input handling

If the user enters something that can't be parsed, the value silently defaults:

TypeDefault on invalid input
int0
float0.0
boolfalse (only "true" is accepted)
strreads whatever was entered as-is

No runtime error is raised. The program continues with the default value.

Note: put() always reads a full line. There is no way to read a single character in v0.1.

String Interpolation

Use curly braces {} inside a string literal to embed variables. The compiler converts each embedded value to a string and concatenates the result at runtime.

im str name = "Shashank"
im int age = 20

out "My name is {name} and I am {age} years old."

Output:

My name is Shashank and I am 20 years old.

Interpolation works with all four types:

TypeInputOutput in string
intim int x = 42"Value: {x}" → Value: 42
floatim float f = 3.14"Pi: {f}" → Pi: 3.140000
boolim bool b = true"Active: {b}" → Active: True
strim str s = "hi""Says: {s}" → Says: hi

Multiple interpolations in one string are supported:

out "{name} is {age} and lives in {city}"

Conditionals — when / otherwise

Shank uses when and otherwise instead of if and else. The body must be indented.

im int score = 85

when score >= 90:
    out "A grade"
otherwise:
    out "Below A"

The otherwise block is optional:

when x > 0:
    out "positive"

Nesting

For multiple conditions, nest when/otherwise blocks:

when x > 100:
    out "huge"
otherwise:
    when x > 50:
        out "medium"
    otherwise:
        out "small"

Compound conditions

when score >= 90 and passed == true:
    out "Excellent"
Note: Shank does not have elif in v0.1. Use nested when/otherwise blocks instead.

Loops — repeat

repeat iterates a fixed number of times. The loop counter is bound to the variable after as, starting from 0.

repeat 5 as i:
    out i

Output:

0
1
2
3
4

The count can be any integer expression:

im int n = 3
repeat n * 2 as idx:
    out "iteration {idx}"

Using the loop variable

The loop variable is an int available inside the body:

mut int total = 0
repeat 10 as i:
    total = total + i
out total        # prints 45

Nesting

repeat 3 as i:
    repeat 3 as j:
        out "{i},{j}"
Note: repeat is the only loop construct in Shank v0.1. There is no while or for-in.

Defining Functions

Functions are defined with def or fn. Both are identical.

def greet(name: str):
    out "Hello, {name}!"

greet("world")

Parameter types

Two styles are supported:

# Python style — name: type
def add(a: int, b: int):
    return a + b

# C style — type name
fn add(int a, int b):
    return a + b

If you omit the type entirely, it defaults to int:

def double(x):
    return x * 2

Functions currently support up to 4 parameters (Windows x64 ABI register limit).

Return Values

The return type does not need to be declared. The compiler infers it by analyzing the return expression.

def square(n):
    return n * n       # inferred as int

out square(7)         # prints 49

You can still declare it explicitly with the arrow syntax:

def square(n) -> int:
    return n * n

Functions without a return statement return none.

Forward Calls

Functions can be called before they are defined in the source file. The compiler resolves all function signatures in a first pass before executing any top-level code.

out square(5)    # works — prints 25

def square(n):
    return n * n

Top-level statements and function definitions can be mixed freely. Functions are hoisted; top-level code runs in order.

out add(2, 3)      # prints 5

im int x = 10
out add(x, 5)      # prints 15

def add(a, b):
    return a + b
Note: Recursive functions work. The compiler does not detect infinite recursion.

Type Casting — ->

The arrow operator -> converts a value from one type to another.

im float pi = 3.14159
im int rounded = pi -> int
out rounded     # prints 3

Supported conversions

FromToBehavior
floatintTruncation toward zero. 3.7 → 3, -2.9 → -2
intfloatExact conversion
intstrDecimal string. 42 → "42"
floatstrDecimal string. 3.14 → "3.140000"
boolstr"true" or "false"

Casts can appear inline:

im float price = 9.99
out "Whole part: {price -> int}"

Type Checking

Shank enforces types at compile time. Mismatches are caught before any executable is produced.

im int x = "hello"
# error: Type mismatch: expected 'int', got 'str'.
im int a = 5
im str b = "hi"
out a + b
# error: Invalid operands to binary operator: 'int' and 'str'.
im int x = 3.14
# error: Type mismatch: expected 'int', got 'float'.

im int x = 3.14 -> int    # correct — explicit cast

The type checker runs automatically during shank build and shank run. Run it standalone with shank check myfile.sk.

Note: Comparison operators require both operands to be the same type. Logical operators (and, or, not) only accept bool operands.
