The Shank Tutorial

This document provides a formal introduction to the core language features of Shank version 0.1.

1. Execution Model

Shank executes scripts from the top level downwards. Unlike C or Java, a designated main entry point is not required for script execution. Statements at the root level of a file are evaluated sequentially.

# Top-level statements are valid
im int x = 10
out "Value of x is {x}"

If functions are declared within the same file, the Shank compiler resolves them in a prior pass. This allows functions to be invoked before their lexical definition.

2. Functions

Functions are defined using the def keyword. A function block must be indented using 4 spaces or 1 tab. The syntax resembles Python but incorporates static typing semantics.

2.1 Parameter Types

Parameter types may be specified explicitly using the name: type syntax. If a parameter type is omitted, the compiler infers the type as int by default.

def add(a, b):
    return a + b

def concat(s1: str, s2: str):
    return s1 + s2

2.2 Return Type Inference

The return type of a function does not need to be declared. The compiler infers the return type by statically analyzing the return expressions within the function body. If the function does not contain a return statement, the inferred type is none.

3. Control Flow

Shank replaces traditional if/else branching with when/otherwise blocks for conditional execution.

3.1 Conditionals

A when block executes if its condition evaluates to true. The otherwise block acts as the fallback execution path.

im int age = 20

when age >= 18:
    out "Adult"
otherwise:
    out "Minor"

3.2 Repetition

Iterative loops over a bounded integer range are handled by the repeat construct. This evaluates an integer expression and binds the current iteration index to the specified identifier.

# Iterates 5 times, i takes values 0 through 4
repeat 5 as i:
    out i

4. Type Casting

Explicit type conversion is performed using the -> operator. The expression on the left is cast to the type identifier on the right.

im float pi = 3.14159
im int rounded = pi -> int

Casting behavior follows standard C truncation rules for floating-point to integer conversions.
