Shank Documentation

Language reference for Shank v0.1 — a statically typed, compiled language with Python-like syntax.

Overview

Shank compiles .sk source files directly to x86-64 machine code via NASM and GCC. The output is a standalone Windows executable — no runtime or interpreter needed.

The compiler toolchain is invoked through the shank CLI:

# compile and run
shank run myfile.sk

# compile to .exe only
shank build myfile.sk -o myfile.exe

# type-check without compiling
shank check myfile.sk

Hello World

Shank programs execute top-level statements from top to bottom. There is no required main function, no imports, no boilerplate.

out "Hello, World!"

Save this as hello.sk and run it:

$ shank run hello.sk
Hello, World!

Variables

Variables are declared with a mutability keyword followed by a type and a name.

Immutable variables — im

Immutable variables cannot be reassigned after their initial value is set.

im int age = 25
im str name = "Shashank"
im float pi = 3.14159
im bool active = true

Mutable variables — mut

Mutable variables can be reassigned freely.

mut int counter = 0
counter = counter + 1
counter = 99
Note: Attempting to reassign an im variable will produce a compile-time error.

Output — out

out prints a value to standard output followed by a newline. It works with all four types.

out 42
out "hello"
out 3.14
out true

You can also print the result of any expression:

im int x = 10
out x * 2       # prints 20

Input — put

put prints a prompt string, reads a line from stdin, and returns the value parsed as the type of the variable it is assigned to.

im int age = put("Enter your age: ")
im str name = put("Enter your name: ")
im float height = put("Height in meters: ")
im bool agree = put("Agree? (true/false): ")

The compiler determines which runtime input function to call based on the declared type of the receiving variable. If the user provides invalid input (e.g. letters when int is expected), the value defaults to 0, 0.0, or false depending on type.

String Interpolation

Use curly braces {} inside a string literal to embed expressions. The compiler converts each embedded expression to a string and concatenates the result at runtime.

im str name = "Shashank"
im int age = 20

out "My name is {name} and I am {age} years old."

Output:

My name is Shashank and I am 20 years old.

Interpolation supports int, float, bool, and str expressions.

Conditional — when / otherwise

Shank uses when and otherwise instead of if and else. The body is indented.

im int score = 85

when score >= 90:
    out "A grade"
otherwise:
    out "Below A"

The otherwise block is optional.

when x > 0:
    out "positive"

Nesting works as expected:

when x > 100:
    out "big"
otherwise:
    when x > 50:
        out "medium"
    otherwise:
        out "small"

Loops — repeat

repeat iterates a fixed number of times. The loop counter is bound to the variable after as, starting from 0.

repeat 5 as i:
    out i

Output:

0
1
2
3
4

The count can be any integer expression:

im int n = 3
repeat n * 2 as idx:
    out "iteration {idx}"

Defining Functions

Functions are defined with the fn or def keyword. Both work identically.

def greet(name: str):
    out "Hello, {name}!"

greet("world")

Parameter types

You can declare parameter types in two styles:

# Python style — name: type
def add(a: int, b: int):
    return a + b

# C style — type name
fn add(int a, int b):
    return a + b

If you omit the type entirely, it defaults to int:

def double(x):
    return x * 2

Functions currently support up to 4 parameters.

Return Values

You do not need to declare the return type. The compiler infers it by analyzing the return statement.

def square(n):
    return n * n       # inferred as int

out square(7)         # prints 49

If you want to be explicit, you can still annotate it with the arrow syntax:

def square(n) -> int:
    return n * n

Functions without a return statement return none.

Forward Calls

Functions can be called before they are defined. The compiler resolves all function signatures in a first pass before executing any code.

out square(5)    # works — prints 25

def square(n):
    return n * n

Type Casting — ->

The arrow operator -> converts a value from one type to another.

im float pi = 3.14159
im int rounded = pi -> int
out rounded     # prints 3

Supported conversions:

FromToBehavior
floatintTruncation (rounds toward zero)
intfloatExact conversion
intstrDecimal string representation
floatstrDecimal string representation
boolstr"true" or "false"

Type Checking

Shank enforces types at compile time. You will get clear errors before any executable is produced.

im int x = "hello"
# error: Type mismatch: expected 'int', got 'str'.
im int a = 5
im str b = "hi"
out a + b
# error: Invalid operands to binary operator: 'int' and 'str'.

The type checker runs automatically during shank build and shank run. You can also run it standalone with shank check.

Note: Comparison operators require both operands to be the same type. Logical operators (and, or, not) only accept bool operands.
